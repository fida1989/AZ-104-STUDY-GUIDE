Exam Domains — Detailed Guide

20–25%

Manage Azure identities and governance

Master Microsoft Entra ID, Azure RBAC, resource organization, governance controls, policy, locks, tags, subscriptions, management groups, and cost governance.

Manage Microsoft Entra users and groups

  • Create cloud users and invited external users. Know when to use member vs guest accounts.
  • Manage group types: security groups for permissions and Microsoft 365 groups for collaboration.
  • Use dynamic membership rules when users/devices should join groups automatically based on attributes.
  • Assign licenses directly or through groups. Group-based licensing is easier to manage at scale.
  • Configure SSPR so users can reset passwords safely. Know authentication methods and registration requirements.

Manage access to Azure resources

  • Authentication proves who the user is. Authorization decides what the user can do.
  • Azure RBAC assignment = security principal + role definition + scope.
  • Scopes from broad to narrow: management group, subscription, resource group, resource. Use the smallest practical scope.
  • Built-in roles include Owner, Contributor, Reader, User Access Administrator, Storage Blob Data Contributor, and Virtual Machine Contributor.
  • Deny assignments are rare for admins and often created by Azure managed apps or policy.

Subscriptions and governance

  • Management groups organize subscriptions for policy and RBAC inheritance.
  • Resource groups are lifecycle containers. Resources in one RG can be in different regions, but the RG metadata has a region.
  • Tags help cost reporting, ownership, environment labels, and automation. Tags do not enforce rules by themselves.
  • Azure Policy can audit, deny, append, modify, or deploy settings. Policy is for compliance, not user permissions.
  • Locks protect resources from accidental delete/write. A lock can block actions even when RBAC allows them.
  • Budgets and cost alerts notify; they do not automatically stop all resources. Azure Advisor gives optimization recommendations.

15–20%

Implement and manage storage

Focus on storage accounts, redundancy, security, Blob Storage, Azure Files, access control, lifecycle management, backup, and migration tools.

Configure storage access

  • Access keys give broad account-level access. Rotate keys and avoid sharing them.
  • SAS gives limited delegated access by permission, resource, IP, protocol, start time, and expiry.
  • Stored access policies are useful because you can revoke or change SAS behavior centrally.
  • Microsoft Entra ID + RBAC is preferred for many Blob and Azure Files access scenarios.
  • Storage firewalls can restrict access by public network, selected networks, private endpoints, and trusted Azure services.

Storage accounts and redundancy

  • LRS: 3 copies in one datacenter. Cheapest, lowest resilience.
  • ZRS: copies across availability zones in one region. Good for zone failure protection.
  • GRS/GZRS: async copy to paired region for regional disaster protection.
  • RA-GRS/RA-GZRS: read access to the secondary region.
  • Object replication copies block blobs between storage accounts. Encryption at rest is enabled by default.

Azure Files and Blob Storage

  • Azure Files provides managed SMB/NFS file shares and can integrate with identity-based access.
  • Blob Storage is object storage for unstructured data such as images, backups, logs, and media.
  • Blob access tiers: hot, cool, cold, archive. Archive has low storage cost but requires rehydration.
  • Soft delete protects against accidental deletion. Versioning protects previous blob versions.
  • Lifecycle management can move blobs between tiers or delete data automatically based on rules.
  • AzCopy is command-line copy/sync. Storage Explorer is GUI-based storage management.

20–25%

Deploy and manage Azure compute resources

Learn VMs, scale sets, App Service, containers, deployment automation, availability, disks, images, and admin operations.

ARM templates and Bicep

  • ARM templates and Bicep define Azure resources as infrastructure as code.
  • Bicep is simpler syntax that transpiles to ARM JSON.
  • Know how to read parameters, variables, resources, outputs, dependencies, and deployment scope.
  • You may need to modify existing templates rather than write from zero.
  • Exports from the portal can help create a starting ARM template, but exported templates often need cleanup.

Virtual machines

  • VM size controls CPU, memory, performance, and cost.
  • Managed disks include Standard HDD, Standard SSD, Premium SSD, and Ultra Disk.
  • Availability sets protect from fault/update domains inside a datacenter. Availability zones protect across separate datacenters in a region.
  • VM Scale Sets provide scalable identical VM instances.
  • Custom Script Extension, Run Command, cloud-init, and Desired State Configuration can help configure VMs.
  • Azure Bastion enables secure RDP/SSH without public IP exposure.

App Service and containers

  • App Service is managed hosting for web apps, APIs, and background web jobs. It uses an App Service plan for compute.
  • Deployment slots support staging/production swap and safer releases.
  • Scale up changes plan size. Scale out adds instances.
  • Azure Container Registry stores container images.
  • Azure Container Instances runs simple containers quickly without Kubernetes.
  • Azure Container Apps is serverless/containerized microservices with scaling. AKS is managed Kubernetes and more advanced.

15–20%

Implement and manage virtual networking

This is one of the most practical areas: VNets, subnets, NSGs, peering, routing, DNS, private access, load balancing, and troubleshooting.

VNets, subnets, and IPs

  • VNets provide private address space in Azure. Address ranges must not overlap when peering or connecting networks.
  • Subnets segment a VNet. Some services require delegated subnets.
  • Public IPs can be Basic or Standard. Standard is secure by default and zone-aware.
  • Private IPs can be dynamic or static. Static is common for appliances, DNS, and stable endpoints.
  • DNS can use Azure-provided DNS, custom DNS servers, private DNS zones, and public DNS zones.

Security and routing

  • NSGs filter traffic by source, destination, port, protocol, direction, priority, and action. Lower priority number wins.
  • ASGs let you group VM NICs logically and use those groups inside NSG rules.
  • Route tables contain user-defined routes. UDRs can send traffic through NVAs, firewalls, or custom next hops.
  • Service endpoints keep traffic to supported services on Azure backbone but still use the service public endpoint model.
  • Private endpoints provide a private IP in your VNet for supported PaaS services.

Connectivity and load balancing

  • VNet peering connects VNets through Microsoft backbone. Peered VNets cannot have overlapping address ranges.
  • VPN Gateway connects on-premises networks using encrypted tunnels. ExpressRoute uses private dedicated connectivity.
  • Azure Load Balancer works at layer 4. Application Gateway works at layer 7 and supports WAF. Traffic Manager is DNS-based global routing.
  • Network Watcher tools: IP flow verify, next hop, effective security rules, connection troubleshoot, packet capture, and connection monitor.

10–15%

Monitor and maintain Azure resources

Know Azure Monitor, alerts, Log Analytics, KQL basics, insights, backup, disaster recovery, update management, and resource health.

Azure Monitor and alerts

  • Metrics are numerical time-series data. Logs are detailed records queried with KQL in Log Analytics.
  • Alerts can trigger action groups such as email, SMS, webhook, ITSM, automation, or functions.
  • Activity Log records control-plane operations such as creating, updating, deleting, and role changes.
  • Diagnostic settings send logs and metrics to Log Analytics, storage accounts, or Event Hubs.
  • Workbooks and dashboards help visualize monitoring data.

Insights and troubleshooting

  • VM Insights monitors VM performance and dependencies. Container Insights helps with AKS/container workloads.
  • Network Insights and Network Watcher help troubleshoot connectivity and traffic flow.
  • Azure Service Health provides service incident and maintenance information. Resource Health gives resource-specific health state.
  • Use KQL filters like where, summarize, project, order by, and time ranges for basic log analysis.

Backup and recovery

  • Azure Backup protects data and workloads. Recovery Services vaults commonly store VM backup data.
  • Backup policies define schedule, retention, and backup frequency.
  • Restore options can include full VM restore, disk restore, file recovery, and alternate location restore depending on workload.
  • Azure Site Recovery replicates workloads for disaster recovery and failover, not normal backup.
  • Know RPO/RTO conceptually: RPO is acceptable data loss window; RTO is acceptable recovery time.

Service Comparison Cheat Sheets

RBAC vs Policy

RBAC answers who can do what. Policy answers what configurations are allowed.

Tag vs Policy

Tags label resources. Policy can require or inherit tags.

Delete lock vs Read-only lock

Delete lock blocks deletion. Read-only lock blocks updates and deletion.

LRS vs ZRS

LRS protects inside one datacenter. ZRS protects across zones in one region.

GRS vs RA-GRS

GRS replicates to paired region. RA-GRS allows read from secondary.

SAS vs Access key

SAS is limited and time-bound. Access key is broad and powerful.

Blob vs Azure Files

Blob stores objects. Azure Files provides SMB/NFS shares.

Hot vs Cool vs Archive

Hot is frequent access. Cool/Cold are less frequent. Archive is offline until rehydrated.

Availability set vs Zone

Set protects from rack/update maintenance. Zone protects from datacenter failure.

Scale up vs Scale out

Scale up means bigger SKU. Scale out means more instances.

ACI vs Container Apps vs AKS

ACI is simplest. Container Apps is serverless microservices. AKS is Kubernetes.

Load Balancer vs App Gateway

Load Balancer is layer 4. Application Gateway is layer 7.

Private endpoint vs Service endpoint

Private endpoint gives private IP. Service endpoint secures service access from subnet.

VPN Gateway vs ExpressRoute

VPN uses encrypted internet tunnel. ExpressRoute uses private provider connectivity.

Metrics vs Logs

Metrics are numeric time series. Logs are detailed queryable records.

Backup vs Site Recovery

Backup restores data. Site Recovery replicates/fails over workloads.
