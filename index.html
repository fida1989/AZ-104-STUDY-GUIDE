How to use this guide

AZ-104 is not a pure theory exam. Read the notes, then practice in the Azure portal, CLI, and PowerShell. Focus on what an administrator does daily: configure access, secure resources, deploy compute, manage storage, troubleshoot networking, monitor health, and protect workloads.

  • First pass: understand concepts and service choices.
  • Second pass: do hands-on labs from the checklist.
  • Final pass: answer all 100 questions and review explanations.

Exam Domains

Manage Azure identities and governance 20–25%

Learn Microsoft Entra users, groups, external users, and self-service password reset. Know the difference between authentication and authorization. Microsoft Entra ID authenticates identities. Azure RBAC authorizes actions on Azure resources. RBAC assignment has three parts: security principal, role definition, and scope. Scopes can be management group, subscription, resource group, or resource. Use least privilege and assign permissions at the smallest practical scope.

Governance includes subscriptions, management groups, resource groups, tags, locks, Azure Policy, budgets, alerts, and Advisor recommendations. Tags organize cost and ownership, but they do not enforce rules. Azure Policy can deny, audit, append, or deploy settings. Locks protect resources from accidental changes or deletion. Budgets and cost alerts help track spending but do not automatically stop usage.

Implement and manage storage 15–20%

Know storage account types, replication choices, access methods, and blob/file features. LRS keeps three copies in one datacenter. ZRS spreads data across availability zones. GRS replicates to a paired region. RA-GRS adds read access to the secondary region.

For access, know account keys, SAS, stored access policies, Microsoft Entra-based access, and firewall rules. SAS is preferred for limited delegated access. Stored access policies make SAS easier to revoke. Azure Files provides SMB/NFS file shares. Blob Storage provides object storage for unstructured data. Blob tiers include hot, cool, cold, and archive. Lifecycle management can move or delete blobs automatically. Soft delete, snapshots, and versioning help recover data.

Deploy and manage Azure compute resources 20–25%

Compute includes VMs, VM Scale Sets, availability sets, availability zones, App Service, Container Instances, Container Apps, and Container Registry. Know how to choose the right service. VMs give maximum control. VMSS is for scalable identical VMs. App Service is managed web hosting. ACI is simple serverless container execution. Container Apps is better for event-driven microservices and scaling. ACR stores container images.

For VMs, understand sizes, disks, encryption at host, movement between resource groups/subscriptions/regions, availability options, and scale sets. For deployment automation, understand ARM templates and Bicep at a reading/modification level. For App Service, know App Service plans, scaling, custom domains, TLS certificates, backup, networking, and deployment slots.

Implement and manage virtual networking 15–20%

Networking is a major hands-on area. Know VNets, subnets, IP addressing, public IPs, NSGs, ASGs, route tables, UDRs, VNet peering, Azure DNS, Load Balancer, Bastion, service endpoints, private endpoints, Network Watcher, and connectivity troubleshooting.

NSGs filter traffic using source, destination, port, protocol, direction, priority, and action. Lower priority numbers are evaluated first. ASGs help group VMs logically inside NSG rules. VNet peering connects VNets over Microsoft backbone, but address spaces must not overlap. Private endpoints provide private IP access to PaaS services. Service endpoints keep traffic on Azure backbone but do not give the service a private IP in your VNet.

Monitor and maintain Azure resources 10–15%

Azure Monitor collects metrics and logs. Metrics are numerical time-series data. Logs are queryable records stored in Log Analytics workspaces. Alerts can use metrics or logs and trigger action groups. VM Insights, Storage Insights, and Network Insights help observe common resources. Network Watcher helps troubleshoot connectivity, NSG flow, next hop, IP flow verify, packet capture, and connection monitor.

Backup and recovery includes Recovery Services vaults, Backup vaults, backup policies, restore operations, Azure Backup, and Azure Site Recovery. Azure Backup protects data and workloads. Azure Site Recovery is for disaster recovery and failover to another region. Know the difference between backup, restore, replication, and failover.

Service Comparison Cheat Sheets

RBAC vs Policy

RBAC controls who can do actions. Policy controls what resources/configurations are allowed.

Lock vs RBAC

RBAC grants permissions. Locks can still block delete/write actions even when RBAC allows them.

Private endpoint vs Service endpoint

Private endpoint gives a private IP for PaaS access. Service endpoint extends VNet identity to PaaS but keeps public endpoint model.

Availability set vs Availability zone

Availability set protects from rack/update-domain failure inside a datacenter. Zones protect across separate datacenters in a region.

Azure Backup vs Site Recovery

Backup restores data. Site Recovery replicates workloads for disaster recovery/failover.

Hot/Cool/Archive

Hot for frequent access. Cool for infrequent access. Archive for lowest cost but delayed retrieval.

ACI vs Container Apps vs AKS

ACI is simplest single-container run. Container Apps is serverless microservices. AKS is managed Kubernetes and more complex.

Metrics vs Logs

Metrics are lightweight numeric time-series. Logs are detailed queryable records using KQL.
